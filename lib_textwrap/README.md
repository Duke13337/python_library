# textwrap — утилита форматирования текста

Модуль **textwrap** из стандартной библиотеки Python даёт функции для форматирования текста: перенос строк по заданной ширине, нормализация отступов, префиксы и обрезка длинного текста. В этом проекте на его основе сделана рабочая утилита: она читает текстовые файлы из папки `data`, форматирует их и выводит результат в консоль или сохраняет в файлы.

---

## 1. Краткое описание инструмента

**textwrap** — стандартный модуль Python для работы с текстом. Он умеет разбивать длинные строки на строки заданной ширины (по словам, без разрыва слов), убирать общий отступ из многострочного текста (`dedent`), добавлять префикс к каждой строке и сокращать текст до заданной длины с многоточием. Зависимостей нет, подходит для консольных утилит и простой генерации текстовых отчётов.

---

## 2. Цель изучения

- Освоить возможности textwrap на практике: `wrap()`, `fill()`, `dedent()`, `shorten()`, `TextWrapper`.
- Понять, как стандартная библиотека закрывает задачу форматирования текста без самописного кода.
- Реализовать прикладную задачу (форматирование файлов из папки) и закрепить работу с путями, аргументами командной строки и структурой проекта.

---

## 3. Описание реализованной задачи

Реализована утилита, которая:

- Читает текстовые файлы из указанного файла или из папки (в т.ч. по умолчанию — папка **data**).
- Форматирует текст с заданной шириной строки (`-w`), с сохранением параграфов (`--preserve-paragraphs`), с нормализацией отступов (`--dedent`), с префиксом (`--prefix`) или с обрезкой (`--shorten`).
- Выводит результат в консоль или сохраняет в указанную папку/файл (`-o`, `--output-file`).

Используются только стандартная библиотека (textwrap, argparse, pathlib) и структура проекта: логика форматирования в `formatter.py`, работа с файлами в `file_handler.py`, интерфейс командной строки в `cli.py`, точка входа — `main.py`.

## 5. Общие выводы

- **textwrap** удобен для форматирования текста в консольных утилитах и при генерации простых текстовых отчётов; не требует установки пакетов.
- Основные плюсы: предсказуемое поведение, перенос по словам, dedent и префиксы «из коробки».
- Ограничения: нет переноса по слогам и сложной типографики; для таблиц или разметки нужны другие инструменты.
- В рамках проекта удалось собрать рабочую программу с разделением на модули и удобным запуском из IDLE одним нажатием Run.
